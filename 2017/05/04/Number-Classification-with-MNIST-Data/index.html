Number Classification with MNIST Data

05-04-2017

See GitHub for Python code.

Recently, my friends and I are trying to train a neural network that can help companies look for venture capital firms. In practice, for a company who is seeking potential investors, we can offer a list of VCs in probability order.

Considering that there are around 6000 investors and 7000 companies in our train data, we start from another simple model: for each handwritten number, offer a list of numbers in probability order.

My solution is that for each picture of written number, we merge it with a classification number picture (0-9), then learn whether these two pictures mean the same number. As shown below, data of written number 5 is merged with data of classification number 5, and the output is a probability distribution, for example (0.9997, 0.0003) (with Softmax classifier, (1,0) indicates that written and classification number is same, (0,1) otherwise.)



For each MNIST picture, we check 10 times (0-9), and get a list of probability distribution. It shows that this MNIST picture most likely means 5.



Now the testing accuracy of this model is around 90%, much lower than record with 99.7%+ accuracy. However, I have not modified my hyperparameters, and this is just a simple demo.
